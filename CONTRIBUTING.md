# Contributing

## Общий принцип

Проект — это personal LMS с жесткими guardrails.

Перед любыми изменениями важно:
- не расширять scope;
- не превращать проект в generic LMS;
- не добавлять platform complexity без причины.

См.:
- docs/product/PROJECT_GUARDRAILS.md

## Как внести изменения

1. Fork репозитория
2. Создать branch
3. Сделать изменения
4. Открыть Pull Request

## Что особенно важно

- изменения должны усиливать learning flow
- не ломать execution-first модель
- не добавлять лишнюю архитектуру
