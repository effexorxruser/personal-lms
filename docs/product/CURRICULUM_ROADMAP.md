# Curriculum Roadmap — Python Backend Developer + AI-enabled Products (6 месяцев)

## Назначение документа

Документ фиксирует практический 6-месячный маршрут обучения в `personal-lms`.

Цель маршрута: провести learner'а от нуля до уровня, где он может самостоятельно собирать небольшие backend + AI-enabled продукты и поддерживать их в рабочем состоянии.

Принцип маршрута: **не "что прочитать", а "что сделать руками"**.

---

## Глобальная рамка маршрута

- Формат: execution-first, source-backed, Russian-friendly wrapper.
- Длительность: 6 месяцев (24 недели).
- Режим: каждый блок заканчивается checkpoint-проектом.
- Результат: набор демонстрируемых артефактов (код, API, тесты, мини-продукты, CI, AI-интеграции).

### Распределение по времени

- **Block 0**: 1 неделя
- **Block 1**: 2 недели
- **Block 2**: 5 недель
- **Block 3**: 4 недели
- **Block 4**: 6 недель
- **Block 5**: 3 недели
- **Block 6**: 3 недели

Итого: 24 недели.

---

## Block 0 — Technical English for Backend (1 неделя)

### 1) Цель

Снять блокер "не понимаю англоязычную документацию" и научить learner'а читать docs целевыми фрагментами без полного перевода.

### 2) Навыки

После блока learner умеет:

- извлекать из документации action items для текущей задачи;
- читать error messages и находить в них ключевые сигналы;
- вести персональный glossary терминов backend/AI;
- формулировать короткие вопросы к AI helper по docs-тексту и ошибкам.

### 3) Core источники (из SOURCE_STACK)

- Python docs / Python Tutorial;
- MDN HTTP (базовые термины);
- FastAPI docs (структура документации);
- OpenAI docs (формат reference/guide).

### 4) Практика (обязательно)

- разобрать 5 коротких doc-отрывков и выписать: цель, вход, выход;
- разобрать 10 типовых терминов (`request`, `response`, `payload`, `endpoint`, `validation` и т.д.);
- взять 3 сообщения об ошибках Python/FastAPI и написать: "что это значит" + "следующее действие";
- сделать мини-упражнение "что читать / что игнорировать" по одной странице docs.

### 5) Artifact (обязательно)

- `tech_english_playbook.md` (личная шпаргалка по чтению docs);
- `error_translation_log.md` (разбор ошибок → действия).

### 6) Реальная ценность

Learner перестает стопориться на английском и может самостоятельно продолжать практику без постоянного внешнего перевода.

### Checkpoint project

**Doc Navigator Kit**: мини-набор шаблонов для чтения документации (термины, паттерн разбора ошибки, шаблон вопроса к AI helper).

## Block 1 — Environment + Git + Terminal + First Execution (2 недели)

### 1) Цель

Собрать рабочую среду и закрепить базовый reproducible workflow: terminal + Git + первый рабочий репозиторий.

### 2) Навыки

После блока learner умеет:

- уверенно запускать базовые команды terminal;
- создавать/клонировать репозиторий, делать commit, branch, pull request;
- оформлять небольшие изменения как проверяемые итерации;
- использовать `.venv` и запускать Python-скрипты в контролируемом окружении.

### 3) Core источники (из SOURCE_STACK)

- GitHub Skills;
- terminal/environments reference-материалы;
- Python docs (venv, запуск скриптов);
- VS Code documentation.

### 4) Практика (обязательно)

- инициализировать репозиторий и сделать серию атомарных commit;
- пройти минимум 2 упражнения по branch/merge из GitHub Skills;
- создать и активировать `.venv`, установить зависимости из `requirements.txt`;
- написать и запустить 2 простых Python-скрипта через terminal.

### 5) Artifact (обязательно)

- публичный/приватный GitHub repo `backend-learning-lab`;
- история commit с понятными сообщениями;
- `setup.md` с шагами запуска.

### 6) Реальная ценность

Learner получает базовую инженерную дисциплину: может воспроизводимо запускать код и вести изменения через Git.

### Checkpoint project

**CLI Setup Verifier**: скрипт, который проверяет окружение (Python version, venv, наличие файлов) и выводит статус готовности.

## Block 2 — Python + Automation (5 недель)

### 1) Цель

Закрыть прикладной Python-минимум для backend: типы, функции, структуры данных, файлы, модули, обработка ошибок, простая автоматизация.

### 2) Навыки

После блока learner умеет:

- писать CLI-утилиты для локальных задач;
- работать с файлами (read/write), JSON/CSV, аргументами командной строки;
- декомпозировать код на функции/модули;
- обрабатывать ошибки и логировать базовые события.

### 3) Core источники (из SOURCE_STACK)

- Python Programming MOOC;
- Automate the Boring Stuff;
- Python docs / Python Tutorial;
- Python logging documentation.

### 4) Практика (обязательно)

- написать CLI-скрипт обработки данных из файла;
- сделать automation-задачу (batch rename, parsing, report generation);
- реализовать обработку ошибок для некорректного ввода;
- добавить базовый лог выполнения (`info/error`);
- оформить задачи с `definition of done` и коротким self-review.

### 5) Artifact (обязательно)

- `automation_toolkit/` с 3+ рабочими скриптами;
- README-модуль с примерами запуска;
- sample outputs (отчеты/файлы).

### 6) Реальная ценность

Learner может автоматизировать рутинные задачи и писать полезные локальные инструменты, а не только учебные примеры.

### Checkpoint project

**Personal Data Processor CLI**: утилита, которая принимает входные данные, очищает/преобразует их и сохраняет итоговый отчет.

## Block 3 — HTTP + APIs + SQLite (4 недели)

### 1) Цель

Перевести Python-навыки в data/service слой: запросы к API, разбор ответов, сохранение данных в SQLite.

### 2) Навыки

После блока learner умеет:

- объяснить базовые HTTP-концепции (method, status code, headers, body);
- делать HTTP-запросы из Python;
- парсить JSON-ответы и валидировать простые поля;
- сохранять/читать данные из SQLite;
- строить pipeline: API → обработка → SQLite.

### 3) Core источники (из SOURCE_STACK)

- MDN HTTP;
- HTTPX documentation;
- SQLBolt;
- Python `sqlite3` documentation.

### 4) Практика (обязательно)

- выполнить серию GET-запросов к публичному API;
- обработать статус-коды и ошибки сети;
- спроектировать простую SQLite-схему под получаемые данные;
- реализовать импорт данных в SQLite и выборку отчетов.

### 5) Artifact (обязательно)

- `data_collector/` сервис-скрипт;
- SQLite база с наполненными таблицами;
- экспортируемый отчет (csv/json/text).

### 6) Реальная ценность

Learner может строить рабочие data collectors и локальные data pipelines для собственных задач.

### Checkpoint project

**API Data Collector**: скрипт/мини-сервис, который по расписанию/запуску тянет данные из API и сохраняет их в SQLite с повторным запуском без дубликатов.

## Block 4 — FastAPI Backend (6 недель)

### 1) Цель

Собрать полноценный учебный backend-сервис с CRUD, валидацией, слоями и понятной структурой.

### 2) Навыки

После блока learner умеет:

- поднимать FastAPI-приложение и проектировать роуты;
- реализовывать CRUD endpoints;
- валидировать входные данные и корректно возвращать ошибки;
- подключать SQLite и выполнять базовые операции хранения;
- документировать API через OpenAPI/Swagger и примеры запросов.

### 3) Core источники (из SOURCE_STACK)

- FastAPI documentation/tutorial;
- Python docs (структуры/исключения/модули);
- MDN HTTP (статусы и semantics);
- SQLBolt и `sqlite3` docs как data support.

### 4) Практика (обязательно)

- создать FastAPI-проект с несколькими ресурсами (`items`, `tasks`, `notes` или аналог);
- реализовать CRUD + фильтрацию + пагинацию (базовый вариант);
- добавить обработчики ошибок и единый формат ответов;
- написать скрипт seed-данных;
- продемонстрировать работу endpoint-ов через HTTP client.

### 5) Artifact (обязательно)

- рабочий backend API-репозиторий;
- OpenAPI schema + примеры вызовов;
- короткое demo-описание (что умеет сервис).

### 6) Реальная ценность

Learner может самостоятельно собрать и показать backend API как портфолио-артефакт.

### Checkpoint project

**Portfolio Backend API**: API для личного трекера задач/контента с SQLite, CRUD и базовой документацией.

## Block 5 — Testing + Debugging + CI (3 недели)

### 1) Цель

Зафиксировать инженерный стандарт качества: тесты, отладка, базовая автоматическая проверка в CI.

### 2) Навыки

После блока learner умеет:

- писать unit/integration тесты для Python/FastAPI;
- воспроизводить и локализовывать баг;
- использовать logging и диагностический вывод осознанно;
- запускать CI workflow (lint/test) через GitHub Actions.

### 3) Core источники (из SOURCE_STACK)

- pytest documentation;
- Python logging documentation;
- debugging guides/reference для Python tooling;
- GitHub Actions CI fundamentals.

### 4) Практика (обязательно)

- покрыть ключевую бизнес-логику тестами;
- добавить тесты на API endpoint-ы;
- воспроизвести 2 намеренно внесенных бага и задокументировать разбор;
- настроить минимальный CI pipeline (`tests` + базовая проверка качества).

### 5) Artifact (обязательно)

- `tests/` с устойчивыми проверками;
- CI workflow, проходящий на push/PR;
- `debugging_notes.md` с кейсами ошибок и фиксов.

### 6) Реальная ценность

Learner умеет не только "собрать", но и поддерживать код: выявлять регрессии и стабилизировать проект.

### Checkpoint project

**Reliability Upgrade Sprint**: улучшение проекта Block 4 через тестирование, фиксы и CI до состояния "демо без ручной магии".

## Block 6 — AI Integration (OpenAI / tools / structured outputs) (3 недели)

### 1) Цель

Добавить AI как контролируемый engineering-layer к уже работающему backend, без автопилота и без ломки архитектуры.

### 2) Навыки

После блока learner умеет:

- вызывать OpenAI API из backend;
- реализовывать structured outputs для предсказуемого формата ответа;
- использовать function/tool calling для ограниченных действий;
- добавлять moderation/checks на вход/выход AI-функции;
- оценивать качество AI-фичи на простом eval-наборе.

### 3) Core источники (из SOURCE_STACK)

- OpenAI quickstart;
- OpenAI SDK docs;
- Responses API docs;
- Structured Outputs;
- Function Calling;
- Moderation;
- Evals;
- OpenAI cookbook.

### 4) Практика (обязательно)

- интегрировать AI endpoint в FastAPI-проект из Block 4;
- реализовать structured schema ответа и валидацию;
- подключить 1 ограниченный tool/function для прикладного действия;
- добавить moderation guardrail;
- провести mini-eval на 10–20 примерах и зафиксировать результаты.

### 5) Artifact (обязательно)

- AI-enabled backend endpoint(ы);
- набор eval-кейсов + отчет;
- demo-сценарий "запрос → AI-ответ → проверяемый результат".

### 6) Реальная ценность

Learner получает прикладной опыт интеграции AI в существующий backend-продукт и может показать это как современный portfolio-signal.

### Checkpoint project

**AI Utility Backend**: мини-сервис, где AI-функция решает конкретную backend-задачу (например, структурирует входной текст в JSON для последующей обработки), с guardrails и eval-отчетом.

## Russian-friendly слой (сквозное требование)

### Формат lesson wrapper (на русском)

Для каждого урока обязательно:

1. **Зачем этот шаг сейчас** (связь с текущим pipeline).
2. **Что читать в источнике** (конкретный section/page).
3. **Что пропустить сейчас** (anti-overload).
4. **Что сделать руками** (1 обязательное действие).
5. **Как понять, что done** (проверяемый критерий).

### Technical English секция в каждом блоке

- 8–15 ключевых терминов блока (EN → RU пояснение);
- паттерны чтения docs (`Overview → Example → Reference`);
- короткие фразы для AI helper: "объясни ошибку", "упрости абзац", "дай чек-лист действий".

### Правило перевода

- Не переводить источники целиком.
- Переводить только минимально необходимый фрагмент для выполнения текущего действия.

---

## Контроль anti-tutorial-hell

Чтобы маршрут не деградировал в consumption-only, каждый lesson обязан иметь:

- обязательное действие (execution);
- submission-формат;
- review-критерий;
- явный next step.

Если действие не определено, lesson не считается authoring-ready.

## Итоговая траектория результата

По завершении 24 недель learner имеет:

- устойчивую базу Python + tooling;
- практику API/SQLite/FastAPI;
- привычку тестирования и CI;
- минимум 6 checkpoint-артефактов;
- финальный AI-enabled backend мини-продукт.

Это соответствует цели `personal-lms`: гарантировать путь к наблюдаемому практическому результату, а не к коллекции пройденных ссылок.
