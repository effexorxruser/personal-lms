# Product Vision

## Назначение документа

Этот документ фиксирует продуктовый смысл `personal-lms`: зачем проект существует, какую проблему решает и каким должен быть результат для пользователя. Он является source of truth для будущих продуктовых решений.

## Суть проекта

`personal-lms` — это personal research-driven LMS для ускоренного входа в Python backend и практическое применение AI в реальных продуктах.

Проект не является витриной курсов, каталогом ссылок или универсальной LMS. Это персональная обучающая система, которая соединяет curated внешние источники, маршрут, практику, progress, review и регулярный выпуск маленьких рабочих артефактов.

## Проблема

В интернете достаточно материалов по Python, backend и AI, но хаотичное обучение часто ломается не из-за нехватки информации, а из-за отсутствия системы:

- слишком много вариантов и источников;
- непонятно, что делать следующим шагом;
- нет обязательства завершать практику;
- легко уйти в tutorial hell;
- прогресс не фиксируется;
- практика нерегулярная;
- нет понятного перехода от чтения к рабочим задачам.

`personal-lms` нужен, чтобы уменьшить decision fatigue и превратить обучение в последовательный рабочий процесс.

## Почему не просто серфинг по интернету

Свободный поиск полезен для research, но плох как основной режим обучения. Он постоянно требует принимать решения: какой курс выбрать, какой урок смотреть, что считать завершенным, когда переходить дальше, как проверить результат.

Платформа должна забирать на себя эти решения там, где они мешают обучению:

- задавать маршрут;
- показывать next step;
- фиксировать progress;
- переводить источник в конкретный lesson wrapper;
- требовать execution, а не только consumption;
- поддерживать stuck flow;
- собирать weekly recap.

## Психологическая мотивация

Продукт должен снижать трение и поддерживать регулярное движение. Пользователь не должен каждый день заново решать, чему учиться. Он должен открыть приложение и увидеть понятный следующий шаг.

Важный эффект: ощущение, что обучение происходит внутри собственной dev learning environment, а не в бесконечной ленте чужих материалов.

## Практическая мотивация

Цель обучения — стать Python-разработчиком, который умеет добавлять AI в реальные продукты.

Практический фокус на 3–6 месяцев:

- Python core;
- Git / GitHub;
- CLI / terminal literacy;
- testing;
- SQL basics;
- HTTP / API basics;
- FastAPI;
- CRUD backend;
- работа с реальными проектами;
- базовые AI-enabled product skills.

Проект не обещает гарантированный доход или трудоустройство. Он должен вести к применимым навыкам, первым прикладным задачам и портфолио из маленьких рабочих артефактов.

## Конечный эффект для пользователя

Пользователь должен:

- понимать текущий маршрут;
- видеть свой progress;
- регулярно выполнять задания;
- отправлять результаты на review;
- иметь механизм помощи, когда застрял;
- накапливать завершенные артефакты;
- постепенно переходить от учебных задач к реалистичным backend + AI product tasks.

## Personal learning OS, а не сайт с уроками

`personal-lms` должен работать как learning OS:

- держит контекст обучения;
- управляет маршрутом;
- связывает источники с практикой;
- хранит runtime state в БД;
- хранит контент в файлах;
- помогает возвращаться к обучению без потери контекста.

Сайт с уроками показывает материалы. Learning OS ведет пользователя через процесс.

## Не зафиксировано

- Конкретная длительность полной программы.
- Финальный состав всех вспомогательных треков.
- Формат будущего деплоя за пределами локального запуска.
