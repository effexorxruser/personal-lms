# Content Strategy

## Назначение документа

Документ фиксирует контентную модель `personal-lms`: курс не пишется с нуля, а строится как source-backed curriculum на основе curated внешних источников.

## Основной принцип

Курс не создается с нуля как авторский учебник.

Платформа использует внешние источники как backbone, но не превращается в каталог ссылок. Внутри `personal-lms` источники получают маршрут, контекст, задания, progress, submissions, review, stuck flow и weekly recap.

## Что такое source-backed curriculum

Source-backed curriculum означает:

- есть curated внешние материалы;
- каждый источник выбран под конкретную учебную цель;
- источник превращается во внутренний lesson wrapper;
- lesson wrapper объясняет, что изучить, что выполнить и как понять, что шаг завершен;
- прогресс и runtime state хранятся в системе, а не во внешнем источнике.

## Почему курс не пишется с нуля

Причины:

- уже существуют сильные открытые материалы;
- написание полного курса с нуля увеличивает scope;
- MVP должен проверять learning flow, а не авторскую методологию с нуля;
- curated модель позволяет быстрее собрать практический маршрут;
- качество повышается за счет опоры на проверенные источники.

## Как используются внешние источники

Внешний источник может использоваться как:

- основной backbone для темы;
- дополнительное объяснение;
- reference;
- practice source;
- project inspiration;
- open source example.

Внутри lesson wrapper нужно фиксировать:

- что именно пройти;
- почему это нужно в маршруте;
- какой результат получить;
- какую задачу выполнить;
- что считается done.

## Curation

Источник должен проходить curation по критериям:

- соответствует цели Python backend + AI in real products;
- не уводит в лишний scope;
- подходит уровню текущего этапа;
- дает практическую пользу;
- доступен и достаточно стабилен;
- не конфликтует с русскоязычным пользовательским слоем платформы.

Если источник хороший, но сложный или избыточный, lesson wrapper должен ограничить его до нужного фрагмента.

## Как избежать каталога ссылок

Платформа не должна просто показывать список внешних материалов.

Обязательные элементы вокруг источника:

- место в маршруте;
- next step;
- практическая задача;
- статус выполнения;
- submission;
- review;
- stuck flow;
- recap.

Ссылка без внутреннего learning action не считается полноценным уроком.

## Уже определенные backbone sources

В качестве источников могут использоваться:

- CS50P;
- Python Programming MOOC;
- Python docs / Python Tutorial;
- FastAPI tutorial;
- freeCodeCamp;
- Stepik;
- roadmap.sh;
- GitHub Skills;
- curated open source repositories;
- другие подходящие открытые источники по мере research/curation.

Финальная карта использования каждого источника может уточняться позже.

## Основной курс и вспомогательные треки

Проект не ограничивается одним-единственным path навсегда.

Основной курс должен вести к Python backend + AI-enabled product skills. Вспомогательные треки могут закрывать отдельные области:

- Git / GitHub;
- CLI / terminal literacy;
- testing;
- SQL;
- HTTP / API;
- AI product skills;
- другие темы, если они усиливают основной маршрут.

Состав вспомогательных треков не полностью зафиксирован и может уточняться позже.

## Хранение контента

Контент должен храниться в файлах. Runtime state должен храниться в БД.

Это разделение важно для:

- прозрачности контента;
- удобства review;
- возможности подключать репозиторий как источник в GPT Projects;
- стабильности runtime state.
