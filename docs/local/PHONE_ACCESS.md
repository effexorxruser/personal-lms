# Доступ с телефона к Personal LMS (LAN / WSL2)

Краткая шпаргалка, если страница не открывается с iPhone или другого устройства в той же Wi‑Fi сети, что и ноутбук.

## Узнать IP ноутбука

- **Windows (GUI):** Параметры → Сеть и Интернет → Wi‑Fi → свойства подключения — IPv4-адрес.
- **В терминале WSL/Linux:** `ip addr` или `hostname -I` (первый адрес часто подходит; в WSL2 адрес может быть «внутренним» для виртуальной сети — см. ниже).

Скрипт `bash scripts/start_local.sh` печатает best-effort LAN URL; при сомнениях сверяйтесь с `ip addr` и адресом Wi‑Fi интерфейса хоста.

## Убедиться, что сервер слушает все интерфейсы

Локальный скрипт запускает:

`uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload`

То есть процесс слушает `0.0.0.0:8000`, а не только `127.0.0.1`. Проверка с машины, где крутится сервер:

```bash
curl -fsS http://127.0.0.1:8000/health
# или
bash scripts/health_local.sh
```

## Firewall

Если с телефона не открывается, а с ноутбука — открывается:

- Проверьте **Windows Firewall** (разрешить входящие на TCP 8000 для профиля частной сети или временно исключить для проверки).
- Антивирус или корпоративный VPN могут блокировать входящие на порт.

## WSL2: порт не виден с телефона

В WSL2 сеть изолирована: телефон обращается к **IP хоста Windows**, а процесс слушает внутри WSL. Иногда достаточно открыть `http://<IP_ноутбука_в_Wi‑Fi>:8000`; если нет — нужен **проброс порта** с Windows на WSL (troubleshooting, не обязательный шаг для всех).

### Пример `netsh interface portproxy` (Windows, от администратора)

Подставьте `<WSL_IP>` — адрес интерфейса WSL2 (часто виден в `ip addr` внутри WSL, например `eth0`).

```text
netsh interface portproxy add v4tov4 listenaddress=0.0.0.0 listenport=8000 connectaddress=<WSL_IP> connectport=8000
```

После проверки правило можно удалить:

```text
netsh interface portproxy delete v4tov4 listenaddress=0.0.0.0 listenport=8000
```

Список правил:

```text
netsh interface portproxy show all
```

Альтернатива без `netsh`: запускать сервер на стороне Windows или использовать SSH-туннель — вне scope этого документа.

## Чеклист вручную

1. Ноутбук и телефон в **одной Wi‑Fi** сети.
2. Сервер запущен: `bash scripts/start_local.sh`.
3. На ноутбуке открывается `http://127.0.0.1:8000`.
4. `bash scripts/health_local.sh` возвращает OK.
5. С телефона: `http://<IP_ноутбука_в_Wi‑Fi>:8000`; при WSL2 при необходимости — portproxy и firewall.
